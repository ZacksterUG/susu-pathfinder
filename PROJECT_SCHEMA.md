## 📊 Источники данных

### API Endpoints

| Endpoint | Описание | Пример |
|----------|----------|--------|
| `GET /integration/map/buildings` | Список всех корпусов | 32 корпуса |
| `GET /integration/map/rooms/buildingId/{id}` | Аудитории корпуса | 247 аудиторий в корпусе 2 |
| `GET /integration/map/BuildingCoordinates/buildingId/{id}/floor/{n}` | Координаты коридоров этажа | Полигон коридора |
| `GET /microgateway/api/Map/Infrastructure/buildingId/{id}` | Лифты и лестницы | Требуется Bearer-токен |

## 🎯 Задача: Кратчайший путь между аудиториями

### Входные данные
1. **Аудитория А** (корпус, этаж, номер)
2. **Аудитория Б** (корпус, этаж, номер)

### Алгоритм решения
```
1. Определить корпуса аудиторий
   ├─ Если разные корпуса → маршрут через улицу
   
2. Построить граф внутри корпуса:
   ├─ Вершины:
   │   ├─ Центроиды аудиторий
   │   ├─ Центроиды лестниц/лифтов
   │   └─ Узловые точки коридоров
   │
   ├─ Рёбра:
   │   ├─ Внутриэтажные (коридоры)
   │   └─ Межэтажные (лестницы/лифты)
   │
3. Применить алгоритм Дейкстры
   └─ Найти кратчайший путь
```

### Граф перемещений
```
                    [Лестница 1]
                   /     |     \
         [Этаж 1]─┘      |      └─[Этаж 2]
            |           |           |
      [Ауд. 101]   [Ауд. 201]  [Ауд. 202]
            |           |           |
      [Ауд. 102]   [Ауд. 203]  [Ауд. 204]
```

## 📝 Примечания

- **Координаты** — локальная система (пиксели плана этажа)
- **Масштаб** — разный для каждого корпуса
- **Лифты** — в данных API помечены как "Лестница" (InfrastructureObjectType)
- **Bearer-токен** — требуется для доступа к инфраструктуре (есть в скриптах)
